# 自动翻译配置

## Summary

翻译作为字幕导出的独立后处理：转录完成生成字幕后，检测到语言不匹配时，用已配置的 LLM 将字幕翻译为目标语言。默认双语输出（原文+译文同一文件），支持按语言分拆字幕，与现有润色-摘要流程互不干扰。

---

## Problem Frame

当前 VoiceGum 的转录结果始终以原始语言输出。用户听非母语内容（如英文播客、日语会议）时，需要将 SRT 文件导出后手动借助外部工具翻译，流程断裂且无法在应用内完成闭环。

现有 LLM 集成已支持润色和摘要，翻译是同一基础设施上的自然延伸——复用已配置的 LLM 提供商和 API 连接，为用户消除「导出 → 外部翻译 → 导回」的手动步骤。

---

## Actors

- A1. **用户**: 配置翻译选项，触发转录流程，获得翻译后字幕
- A2. **LLM 服务**: 接收翻译请求并返回翻译结果（OpenAI / Anthropic / Ollama 兼容接口）

---

## Key Flows

- F1. **自动翻译 — 逐条模式**
  - **Trigger:** 转录完成，翻译开关开启，检测到语言不匹配
  - **Actors:** A1, A2
  - **Steps:** 转录生成字幕 → 检测源语言 ≠ 目标语言 → 逐条发送字幕文本到 LLM 翻译 → 每条译文保留原始时间轴 → 按用户配置生成仅译文或双语 SRT → 自动导出
  - **Outcome:** 双语模式下一个 SRT 文件（含原文+译文），仅译文模式下两个 SRT 文件（原文 + 译文各一）
  - **Covered by:** R1, R2, R3, R4, R5

- F2. **自动翻译 — 整批模式**
  - **Trigger:** 同 F1，翻译模式设为整批
  - **Actors:** A1, A2
  - **Steps:** 转录生成字幕 → 将所有字幕文本合并（附带序号/时间标记）→ 一次发送到 LLM 翻译 → 解析返回的翻译结果回各字幕段 → 保留原始时间轴 → 生成 SRT 导出
  - **Outcome:** 同上
  - **Covered by:** R1, R2, R3, R4, R6

- F3. **用户配置翻译**
  - **Trigger:** 用户在设置中修改翻译选项
  - **Actors:** A1
  - **Steps:** 用户在 General Tab 开关翻译并选目标语言 → 在 LLM Tab 选翻译模式和编辑提示词 → 配置立即持久化
  - **Outcome:** 后续转录流程按新配置执行翻译
  - **Covered by:** R7, R8, R9, R10, R15, R16

---

## Requirements

**翻译触发与语言检测**
- R1. 翻译仅在转录成功完成且有字幕片段时触发，与润色/摘要流程互不干扰
- R2. 语言不匹配判断：本地 SenseVoice 引擎使用其返回的 detected language；在线 ASR 使用用户配置的源语言（`AppPreferences.language`）。当源语言 ≠ 目标语言时触发翻译
- R3. 源语言与目标语言相同时跳过翻译，不产生额外 LLM 调用

**翻译模式**
- R4. 逐条模式：每条字幕片段独立发送 LLM 翻译，译文保留原片段时间轴，支持精确的双语逐条对照
- R5. 整批模式：所有字幕文本合并后一次发送 LLM 翻译，LLM 返回后按原文结构解析回各片段并保留时间轴
- R6. 翻译模式为用户可选配置，默认逐条模式

**翻译输出**
- R7. 翻译结果保留原始字幕的时间轴信息（startMs / endMs）
- R8. 用户可选择输出模式：双语（默认，只生成一个 SRT 文件，每条字幕块内原文与译文并列）或仅译文（生成两个独立 SRT 文件，原文和译文各一个）
- R9. 翻译后 SRT 自动导出行为跟随现有的 `subtitleExportEnabled` 开关

**语言分拆**
- R10. 支持按检测到的语言分拆字幕：开启后，将字幕片段按各自检测到的语言分组，每种语言生成独立的字幕文件。适用于多语混杂场景（如中英双语对话），方便后续分别处理或翻译
- R11. 语言分拆依赖 SenseVoice 片段级语言检测能力，仅本地引擎可用；在线 ASR 无片段级语言信息，此选项不生效

**LLM 翻译实现**
- R12. 复用已配置的 LLM 提供商和 API 连接，新增 `translate` 方法处理翻译请求
- R13. 翻译使用独立的系统提示词，代码中提供默认提示词（中文），用户在 LLM 设置 Tab 中可编辑
- R14. 默认翻译提示词应引导 LLM：保持口语风格、不添加额外解释、按格式要求返回

**UI 配置**
- R15. General 设置 Tab 新增：翻译开关（默认关闭）、输出模式选择（双语/仅译文，默认双语）、目标语言选择器（默认中文）、语言分拆开关（默认关闭）
- R16. LLM 设置 Tab 的「Task Prompts」区域新增：翻译模式选择（逐条/整批）、翻译提示词编辑框
- R17. 目标语言选项至少包含：中文（zh-CN）、英文（en）、日文（ja）、韩文（ko），与现有语言选择器保持一致

**数据持久化**
- R18. 翻译结果（译文文本、翻译所用目标语言）写入 HistoryEntry 持久化
- R19. 所有翻译相关配置通过 `AppPreferences`（UserDefaults）持久化

## Acceptance Examples

- AE1. **Covers R1, R2, R3.** 用户配置源语言为英文、目标语言为中文，翻译开关开启。转录一段英文音频，SenseVoice 返回 detected language = "en"。源语言(en) ≠ 目标语言(zh) → 触发翻译，生成中文译文 SRT。
- AE2. **Covers R2, R3.** 用户配置源语言为中文、目标语言为中文。转录一段中文音频。源语言(zh) = 目标语言(zh) → 跳过翻译，不调用 LLM。
- AE3. **Covers R4.** 逐条模式下，一段 3 条字幕的转录结果分别发送 3 次 LLM 请求，每条译文的时间轴与原片段一致。
- AE4. **Covers R5.** 整批模式下，3 条字幕合并为一次 LLM 请求，返回的译文解析回 3 条并各保留原时间轴。
- AE5. **Covers R8.** 双语模式：生成一个 SRT 文件，每条字幕块为「原文\n译文」。仅译文模式：生成两个 SRT 文件（`*_original.srt` 和 `*_zh.srt`）。
- AE6. **Covers R9.** `subtitleExportEnabled` 为 false 时，翻译仍执行但翻译后的 SRT 不自动导出，用户可在历史记录中查看译文并手动导出。
- AE7. **Covers R2.** 用户使用 Volcano Engine 在线 ASR（无语言检测），源语言配置为英文。转录完成后源语言(en) ≠ 目标语言(zh) → 触发翻译。
- AE8. **Covers R10, R11.** 语言分拆开启，SenseVoice 检测到字幕中有英文和中文片段。输出两个独立 SRT 文件：`*_en.srt` 和 `*_zh.srt`，各自只包含对应语言的字幕片段。在线 ASR 模式下此选项灰掉不生效。
- AE9. **Covers R8.** 默认双语模式下转录英文音频并翻译为中文，生成一个 SRT 文件，每条字幕块为「英文原文\n中文译文」。

---

## Success Criteria

- 用户转录非母语内容后，无需离开应用即可获得翻译字幕
- 逐条翻译模式下译文时间轴与原文精确对应，可直接用于双语视频字幕
- 翻译配置与现有润色/摘要配置互不干扰，用户可按需独立开关
- 整批翻译的上下文质量明显优于逐条翻译（长文本场景下）

---

## Scope Boundaries

- 不涉及实时/流式翻译，仅限转录完成后的后处理
- 不涉及多目标语言同时翻译（一次只翻一种目标语言）
- 不涉及非 LLM 的本地翻译方案
- 翻译流程不与润色/摘要流水线交叉编排

---

## Key Decisions

- 翻译作为独立线路而非插入现有流水线：避免流水线膨胀，用户可按需组合翻译 + 润色，互不阻塞
- 翻译放在字幕生成之后：确保翻译输入是结构化的字幕片段而非原始转录文本，时间轴信息完整
- 逐条为默认模式：更符合字幕翻译的精确对应需求，双语对照场景下逐条是刚需

---

## Dependencies / Assumptions

- 依赖现有 LLM 集成基础设施（`LLMClient` actor、提供商配置、API 连接）
- 依赖现有字幕生成和 SRT 导出能力（`SubtitleFormatter`、`TranscriptionResult.segments`）
- 假设 LLM 返回的整批翻译结果可以通过分段标记可靠地解析回各字幕段
- 假设在线 ASR 场景下用户配置的源语言准确反映音频实际语言（在线 ASR 无自动检测能力）
